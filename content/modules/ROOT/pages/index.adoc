= Red Hat Developer Lightspeed for MTA (Konveyor AI)
:navtitle: はじめに

このワークショップへようこそ！

Legacy Java Japan カンパニーは、長年稼働している Java EE アプリケーション群の保守コストに頭を抱えています。
開発チームはリリースのたびに複雑な依存関係と格闘し、クラウドネイティブへの移行はいつも「次のスプリント」に先送りされてきました。
あなたのミッションは、Red Hat Developer Lightspeed と Migration Toolkit for Applications (MTA / Konveyor AI) を使い、その状況を変えることです。

== このワークショップで学ぶこと

このワークショップを通じて、次のことができるようになります。

* IDE プラグインの設定方法を理解する
* アプリのマイグレーションに必要なステップを把握する
* カスタムルールの作成方法を習得する

== 対象者

このワークショップは、Java アプリケーションの Quarkus 移行に取り組む **開発者** を対象としています。

*経験レベル*: 中級 (Intermediate)

OpenShift の基本操作と Java 開発の経験があると、よりスムーズに学習を進められます。

== 事前準備

ワークショップを開始する前に、次の点を確認してください。

* Java アプリケーション開発の基礎知識
* OpenShift Web コンソールへの基本的な操作経験
* IDE (VS Code または IntelliJ IDEA) の使用経験

特別なインストール作業は不要です。ラボ環境はすでにセットアップされています。

== ラボ環境

このワークショップのラボ環境には、以下のリソースがあらかじめ用意されています。

* OpenShift クラスター (マルチノード構成)
* MTA サーバー (Konveyor AI バックエンド)
* Dev Spaces (ブラウザベースの IDE)
* サンプル Java EE アプリケーション

環境へのアクセス情報は、ラボインターフェースのサイドパネルで確認できます。

== モジュール構成

このワークショップは 4 つのモジュールで構成されています。

[cols="1,2,1",options="header"]
|===
|モジュール |内容 |所要時間

|モジュール 1: Plugin 設定
|IDE に Developer Lightspeed プラグインをインストールし、MTA サーバーへ接続する
|45 分

|モジュール 2: サンプルアプリのマイグレーション
|Java EE のサンプルアプリケーションを Quarkus に移行する手順を実践する
|60 分

|モジュール 3: カスタムルールの追加
|組織固有の移行ルールを作成し、MTA に適用する方法を学ぶ
|60 分

|モジュール 4: サンプルアプリ 2 のマイグレーション
|カスタムルールを使って 2 つ目のアプリケーションを移行する
|75 分
|===

== 所要時間

このワークショップの完了には、合計 **約 4 時間** を見込んでください。

自分のペースで進めてかまいません。各モジュールは独立して設計されているため、途中で休憩を取ることもできます。
