= モジュール 1: Plugin 設定
:source-highlighter: rouge
:toc: macro
:toclevels: 1

toc::[]

Legacy Java Japan カンパニーのシニア開発者として、あなたはチームを代表して大きな挑戦を任されています。数百の Java EE アプリケーションを Quarkus へ移行するプロジェクトのパイロットを担当することになりました。上司からのメッセージは明確です。「手動での移行は限界だ。MTA と Konveyor AI を使って、まずひとつのアプリで移行を自動化してみてほしい。」

プレッシャーを感じながらも、あなたはその可能性に期待しています。まずは開発環境をセットアップし、Konveyor AI プラグインを IDE に接続することから始めましょう。ここで成功すれば、移行プロセス全体が大きく変わります。

このモジュールでは、OpenShift Dev Spaces で開発環境を構築し、Konveyor AI (Developer Lightspeed) プラグインを MTA サーバーに接続するまでの手順を実践します。

== 学習目標

このモジュールを完了すると、以下のことができるようになります:

* OpenShift Dev Spaces にアクセスし、ワークスペースを起動する
* VS Code 拡張機能として Konveyor AI プラグインをインストールし設定する
* MTA サーバーへの接続を確立し、分析が実行可能な状態であることを確認する

== エクササイズ 1: Dev Spaces ワークスペースの起動

Legacy Java Japan カンパニーの移行プロジェクトには、一貫した開発環境が欠かせません。まず OpenShift Dev Spaces にアクセスし、今回のラボで使用するワークスペースを起動します。

. ブラウザで Dev Spaces のコンソールを開きます。URL は Showroom のタブから確認できます:
+
[source,bash,role="execute"]
----
echo "Dev Spaces URL: {devspaces_url}"
----

. Dev Spaces のログイン画面が表示されたら、次の認証情報でログインします:
+
[source,text]
----
ユーザー名: {user}
パスワード: {password}
----

. ログイン後、「Create Workspace」画面が開きます。

. ワークスペース作成画面で、Git リポジトリの URL を入力します。今回のラボ用リポジトリは事前に準備されています:
+
[source,text]
----
{app_1_gitrepo_url}
----

. 「Create & Open」ボタンをクリックしてワークスペースを起動します。ワークスペースの初期化には 1〜2 分かかります。

.. 以下のような画面が出た場合は、「Continue」をクリックしてください。

image::module01-popup_create_devspace.png[Dev Spaces ワークスペース作成時のポップアップ, link=self,window=blank,width=700]

.. ワークスペース起動時に以下の画面がでたら「Trust Publisher & Install」をクリックしてください。続いて同様にディレクトリを信頼することを確認するメッセージが出ますが、信頼するをクリックしてください。

image::module01-popup_started_devspaces.png[Dev Spaces ワークスペース起動時のポップアップ, link=self,window=blank,width=700]

. ワークスペースが起動したら、画面左下の git のブランチ名が「rhsc2025-handson」になっていることを確認します。

image::module01-devspaces-workspace.png[Dev Spaces ワークスペースが起動した画面,link=self,window=blank,width=700]

=== 確認

ワークスペースが正常に起動したことを確認します。Dev Spaces 内のターミナルで次のコマンドを実行します:

[source,bash,role="execute"]
----
echo "ワークスペース: $DEVWORKSPACE_NAME"
pwd
ls -la
----

`/projects/coolstore` ディレクトリが存在し、Java プロジェクトのファイル構成が表示されれば成功です。

== エクササイズ 2: Konveyor AI プラグインのインストールと設定

ワークスペースが起動しました。次は、Migration Toolkit for Applications (MTA) の AI 機能を IDE から利用するために、Konveyor AI プラグインをインストールして設定します。このプラグインが Legacy Java Japan カンパニーの移行加速の鍵になります。

=== 前提条件

* エクササイズ 1 で Dev Spaces ワークスペースが起動していること
* MTA サーバーの URL が手元にあること

=== 手順

. コマンドパレットを開きます (`Ctrl+Shift+P` または `F1`):
+
[source,bash,role="execute"]
----
echo "コマンドパレットのショートカット: Ctrl+Shift+P"
----

. コマンドパレットに `MTA: Configure Solution` と入力し、設定コマンドを選択します。

. 設定画面が開いたら、以下の項目の設定をしてください。

.. General Settings 項目の「Enable Hub」を有効化
.. 「Hub URL」に {mta_ui_internal_url}
.. 「Insecure connection」を有効化
.. Authentication 項目の「Enable authentication」を有効化
.. 「Username」に `admin`を指定し、「Password」に {mta_admin_password} を指定する

. 「Save」ボタンをクリックして設定を保存します。

. プラグインの設定ファイルが正しく作成されたことをターミナルで確認します:
+
[source,bash,role="execute"]
----
cat ~/.vscode-server/data/User/settings.json | grep -i konveyor
----

image::module01-konveyor-plugin-settings.png[Konveyor AI プラグインの設定画面,link=self,window=blank,width=700]

=== 確認

プラグインの設定が正常に完了したことを確認します。

. VS Code のステータスバーを確認します。Konveyor アイコンが表示されていれば、プラグインが有効になっています。

. コマンドパレット (`Ctrl+Shift+P`) を開き、`Konveyor` と入力します。Konveyor 関連のコマンドが複数表示されれば、プラグインが正常にインストールされています:
+
[source,text]
----
MTA: Configure Solution Server Credentials
MTA: Run Analysis
MTA: Open Dtails
...
----

=== よくある問題

**問題**: 設定を保存しても MTA URL が反映されない

→ **解決方法**: VS Code ウィンドウを再読み込みします (`Ctrl+Shift+P` → `Developer: Reload Window`)。

== エクササイズ 3: MTA サーバーとの接続確認

プラグインの設定が完了しました。いよいよ MTA サーバーとの接続を確認します。Legacy Java Japan カンパニーの移行プロジェクトでは、MTA が分析の中心的な役割を担います。接続が確立できれば、いつでも AI 支援による移行分析を開始できます。

=== 手順

. まず、MTA サーバーが稼働していることをコマンドラインから確認します:
+
[source,bash,role="execute"]
----
curl -s -o /dev/null -w "%{http_code}" {mta_url}/hub/applications
----
+
HTTP ステータスコード `200` または `401` が返れば、サーバーは正常に稼働しています (`401` は認証が必要なことを示します)。

. 次に、MTA サーバーへの認証付きアクセスを確認します:
+
[source,bash,role="execute"]
----
curl -s -u {user}:{password} {mta_url}/hub/applications | python3 -m json.tool | head -20
----
+
アプリケーション一覧が JSON 形式で返ってくれば、接続と認証は成功です。

. VS Code のコマンドパレット (`Ctrl+Shift+P`) を開き、`Konveyor: View Issues` を実行します。

. Konveyor のサイドパネルが開き、MTA サーバーからのデータが表示されることを確認します。

. プラグインから簡単な接続テストを実行します。コマンドパレットで `Konveyor: Configure` を選択し、設定画面で「Test Connection」ボタンをクリックします。

. 「Connection successful」または同等のメッセージが表示されれば、すべての設定が正常です。

image::module01-mta-connection-success.png[MTA サーバーへの接続成功画面,link=self,window=blank,width=700]

=== 確認

接続が確立されていることを包括的に確認します。

[source,bash,role="execute"]
----
# MTA サーバーの稼働状態を確認
curl -s -u {user}:{password} {mta_url}/hub/applications | python3 -c "
import json, sys
data = json.load(sys.stdin)
if isinstance(data, list):
    print(f'接続成功: {len(data)} 件のアプリケーションが登録されています')
else:
    print('接続成功: MTA サーバーに接続できました')
"
----

アプリケーション件数が表示されれば、Konveyor AI プラグインから MTA サーバーへの接続が完全に確立されています。これで移行分析を開始する準備が整いました。

=== よくある問題

**問題**: `curl` コマンドが接続タイムアウトになる

→ **解決方法**: MTA の URL が正しいか確認します。また、OpenShift コンソールで MTA の Pod が Running 状態になっているか確認してください:
+
[source,bash,role="execute"]
----
oc get pods -n mta -l app=mta-hub
----

**問題**: 認証エラー (HTTP 403) が返る

→ **解決方法**: ユーザー名とパスワードが正しいか確認します。Showroom のラボ情報から最新の認証情報を取得してください。

== モジュールまとめ

環境のセットアップと接続確認を完了しました。「これは使えそうだ」と感じた瞬間があったなら、それは正しい直感です。

**達成したこと:**

* OpenShift Dev Spaces で一貫した開発環境を構築した
* Konveyor AI (Developer Lightspeed) プラグインをインストールし設定した
