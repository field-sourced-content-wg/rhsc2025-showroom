= ワークショップ詳細
:toc:
:toc-placement: preamble
:icons: font

このページでは、ワークショップのスケジュール、技術要件、環境セットアップ、よくある問題の解決策を説明します。ワークショップ開始前に必ずご確認ください。

== タイミングとスケジュール

=== フルワークショップ (4 時間)

* **Module 1**: Plugin 設定 (45 分)
* **Module 2**: サンプルアプリのマイグレーション (60 分)
* **Module 3**: カスタムルールの追加 (60 分)
* **Module 4**: サンプルアプリ 2 のマイグレーション (75 分)
* **休憩**: 各モジュール間に 10 〜 15 分

=== 短縮版ワークショップ (2 時間)

時間が限られている場合は、次のモジュールに絞って進めることができます。

* **Module 1**: Plugin 設定 (30 分)
* **Module 2**: サンプルアプリのマイグレーション (45 分)
* **Module 3**: カスタムルールの追加 (45 分) - カスタムルールのインポートまで

[NOTE]
====
短縮版では Module 4 (サンプルアプリ 2 のマイグレーション) をスキップします。Module 4 は Module 3 で作成したカスタムルールを前提としているため、個人学習として後から実施することを推奨します。
====

== 技術要件

=== 使用製品とバージョン

このワークショップでは次の Red Hat 製品を使用します。

* Red Hat OpenShift {ocp_version} 以降
* Migration Toolkit for Applications (MTA) 7.0 以降
* Konveyor AI (Developer Lightspeed) - MTA に統合済み
* Red Hat OpenShift Dev Spaces — ブラウザベースの IDE
* Web ブラウザ (Chrome 120 以降、Firefox 120 以降、Safari 17 以降、Edge 120 以降)

=== 環境アクセス

参加者は次の環境にアクセスします。すべてワークショップ当日に提供されます。

* **Red Hat OpenShift クラスター** — マルチノード構成、ワークショップ用に事前設定済み
* **Red Hat OpenShift Dev Spaces** — ブラウザから直接アクセスできる統合開発環境
* **MTA Web コンソール** — OpenShift クラスター上にデプロイ済み
* **ターミナル** — Showroom の Terminal タブから SSH アクセス可能

[IMPORTANT]
====
ローカル PC へのソフトウェアインストールは不要です。すべての操作はブラウザと Showroom の Terminal タブから実行します。
====

=== ネットワーク要件

* インターネット接続 (HTTPS ポート 443)
* OpenShift クラスターへのアクセス (提供される URL を使用)
* GitHub または内部 Git リポジトリへのアクセス (サンプルコードの取得)

== 環境セットアップ

=== Showroom タブの確認

このワークショップでは Showroom の 3 つのタブを使用します。

[cols="1,3",options="header"]
|===
|タブ名 |用途

|Terminal
|コマンド実行、ファイル操作、oc CLI の使用

|OCP Console
|OpenShift Web コンソール — MTA の確認やリソースの監視

|DevSpaces
|Red Hat OpenShift Dev Spaces — IDE プラグインの設定と Java コードの編集
|===

=== ワークショップ開始前チェックリスト

ワークショップを始める前に、次の項目を確認してください。

☐ **OpenShift クラスターへのログイン確認** — 提供されたクレデンシャルでログインできることを確認する +
☐ **DevSpaces が起動していること** — DevSpaces タブを開き、ワークスペースが Ready 状態であることを確認する +
☐ **MTA Web コンソールへのアクセス確認** — OCP Console タブから MTA にアクセスできることを確認する +
☐ **Terminal が応答していること** — Terminal タブでコマンドを実行できることを確認する

=== セットアップ検証

Terminal タブで次のコマンドを実行し、環境が正しくセットアップされていることを確認します。

[source,bash,role="execute"]
----
# OpenShift にログインできているか確認
oc whoami
----

Password: `{bastion_ssh_password}`

[source,bash,role="execute"]
----
# MTA Operator がデプロイされているか確認
oc get pods -n mta
----

[source,bash,role="execute"]
----
# Dev Spaces Operator が稼働しているか確認
oc get pods -n openshift-devspaces
----

次のような出力が表示されれば、環境の準備は完了です。

[source,text]
----
# oc whoami の期待出力例
user1

# oc get pods -n mta の期待出力例 (一部)
NAME                                READY   STATUS    RESTARTS   AGE
mta-ui-xxxxxxxx-xxxxx               1/1     Running   0          10m
mta-hub-xxxxxxxx-xxxxx              1/1     Running   0          10m
----

== トラブルシューティングガイド

=== よくある問題と解決策

**問題**: DevSpaces のワークスペースが起動しない、または Starting のまま停止する +
→ **解決策**: OCP Console タブから `openshift-devspaces` namespace を開き、Pod のログを確認してください。ワークスペースの Pod が `Pending` 状態の場合はリソース不足の可能性があります。ファシリテーターに連絡してください。

**問題**: MTA Web コンソールに接続できない +
→ **解決策**: Terminal タブで `oc get route -n mta` を実行し、Route の URL を確認してください。URL をブラウザに直接貼り付けてアクセスしてみてください。

**問題**: Developer Lightspeed プラグインが DevSpaces に表示されない +
→ **解決策**: DevSpaces の Extensions ビュー (`Ctrl+Shift+X`) で `Konveyor` または `MTA` を検索してください。プラグインが見つからない場合は、Module 1 の手順に従って手動でインストールします。

**問題**: `oc` コマンドで `Unauthorized` エラーが表示される +
→ **解決策**: 次のコマンドで再ログインしてください。クレデンシャルはワークショップ提供資料を参照してください。

[source,bash,role="execute"]
 oc login --server={api_url} --username={user} --password={password}

**問題**: Java サンプルアプリのビルドが失敗する +
→ **解決策**: DevSpaces のターミナルで `java -version` と `mvn -version` を実行し、Java 17 以降と Maven 3.8 以降がインストールされていることを確認してください。DevSpaces のワークスペース定義には必要なツールが含まれているため、通常は自動的に利用可能です。
